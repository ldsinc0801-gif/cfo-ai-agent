# AI CFO 営業レクチャーガイド
## 客先デモ・導入説明の流れ

---

## 1. 導入トーク（3分）

### 伝えるポイント

> 「AI CFOは、中小企業の経営者が**毎日の経理作業**と**経営判断**の両方をAIで効率化するツールです。
> レシートを撮影するだけで仕訳が完了し、その数字をもとに経営シミュレーションまでできます。
> さらに、銀行がどう評価するかもリアルタイムで確認できるので、融資対策にも使えます。」

### 3つの価値を簡潔に

| 価値 | 一言で |
|------|--------|
| 経理の自動化 | レシート撮影 → freee自動登録 |
| 経営の見える化 | 売上・利益・CFをシミュレーション |
| 銀行評価の把握 | 自社の格付を事前に確認 |

---

## 2. デモの流れ

### STEP 1: 会計AI（5分）

**画面：** `http://localhost:3000/agent/accounting`

#### 操作手順

1. 「経営シミュレーター」ではなく、まず**会計AI**から見せる
2. レシート画像（スマホで撮影したもの）をアップロード
3. AIが自動で読み取り → 仕訳データが表示される

#### 説明ポイント

> 「レシートを写真に撮ってアップロードするだけで、AIが勘定科目・金額・税区分を自動判定します。
> 経理の知識がなくても仕訳ができます。」

4. 読み取り結果を確認 → 「freeeに送信」ボタンを押す
5. freeeの画面を開いて、取引が登録されていることを確認

#### よくある質問への回答

- **Q: 読み取り精度は？** → 「Gemini AIを使用しており、印刷されたレシートなら95%以上の精度です。手書きでも読み取れますが、確認画面で修正できます」
- **Q: freee以外は？** → 「現在はfreee対応。弥生CSV出力にも対応しています」
- **Q: 複数枚まとめては？** → 「複数ファイル同時アップロード可能です。動画からの読み取りにも対応しています」

---

### STEP 2: 事業計画AI - KPI目標設定（5分）

**画面：** `http://localhost:3000/plan`  
**タブ：** 「KPI目標・ギャップ分析」

#### 操作手順

1. 年間KPI目標を入力する
   - 目標売上高
   - 目標営業利益
   - 目標営業利益率
   - 目標自己資本比率
   - 目標労働生産性
   - 従業員数

2. freeeの実績が自動取得されるので、目標とのギャップが表示される

#### 説明ポイント

> 「freeeに入力した日々の数字が、自動でKPI目標と比較されます。
> 目標に対して今どの位置にいるかが一目でわかります。
> 赤ならまずい、黄色なら注意、緑なら順調。経営者が毎朝見るべき画面です。」

3. 事業計画書（PDF/CSV）のアップロードも可能
   - AIが自動解析して目標に反映

---

### STEP 3: 経営シミュレーター（10分）★メインデモ

**画面：** `http://localhost:3000/plan`  
**タブ：** 「経営シミュレーター」

#### 3-1. ベースデータの確認

1. 「実績ベース」ボタンが選択されていることを確認
2. 上部に「freee実績データ（直近12ヶ月）」と表示される

> 「freeeに入力されている直近12ヶ月の実績をベースに、これからの1年をシミュレーションします。
> 過去の数字から未来を予測するので、地に足のついた計画が立てられます。」

#### 3-2. シナリオ調整（ここが一番盛り上がる）

左側のスライダーを動かしながら説明する：

| スライダー | 説明トーク |
|-----------|-----------|
| **売上成長率** | 「例えば売上を10%伸ばしたらどうなるか。右にスライドしてみてください」 |
| **原価率** | 「仕入先を変えて原価を5%下げたら？利益がどう変わるか見てみましょう」 |
| **固定費増減** | 「事務所を移転して家賃が上がる場合。固定費+10%にしてみましょう」 |
| **人員増減** | 「社員を2人採用したら？人件費の影響がすぐわかります」 |
| **新規設備投資** | 「500万の機械を買ったら？減価償却が自動計算されます」 |
| **年間借入返済** | 「借入の返済がキャッシュフローにどう影響するか確認できます」 |

#### 3-3. グラフの見方

> 「グレーの点線が今のまま何もしない場合。色付きの実線がシミュレーション結果です。
> 売上を上げても固定費が増えると利益は変わらない、というのが一目でわかります」

#### 3-4. 実績ベース → 計画ベースの切替

1. 「計画ベース」ボタンをクリック
2. KPI目標の数値がベースに切り替わる

> 「さっきは実績ベースでしたが、今度は目標値をベースにシミュレーションします。
> 目標を達成するには、売上を何%伸ばして、コストをどこまで下げる必要があるか。
> スライダーで条件を探すことができます」

#### 3-5. 計画に反映

1. 良いシミュレーション結果が出たら「計画に反映する」ボタンを押す
2. KPI目標に自動反映される

> 「シミュレーションで出した数字を、そのまま事業計画の目標にセットできます。
> 根拠のある計画が、クリック一つで完成します」

---

### STEP 4: ロカベン6指標（5分）

**画面：** 経営シミュレーター内の「ロカベン6指標」セクション

#### 操作手順

1. 業種を選択（大分類 → 小分類）
2. 事業規模を選択（中規模 or 小規模）
3. 6指標のランクが自動表示される

#### 説明ポイント

> 「ロカベンとは、経済産業省が作った企業の健康診断ツールです。
> 同業他社と比べて自分の会社がどの位置にいるかがわかります。
> A〜Eの5段階で、Aが上位、Eが下位です」

#### 6指標の説明

| 指標 | 経営者向け説明 |
|------|-------------|
| 売上増加率 | 「売上が伸びているか」 |
| 営業利益率 | 「本業でちゃんと儲けているか」 |
| 労働生産性 | 「社員一人あたりどれだけ稼いでいるか」 |
| EBITDA有利子負債倍率 | 「借金を何年で返せるか」 |
| 営業運転資本回転期間 | 「お金の回りが良いか」 |
| 自己資本比率 | 「会社の体力があるか」 |

> 「スライダーを動かすと、この指標もリアルタイムで変わります。
> 売上を10%伸ばしたら、業界内でBランクからAランクに上がる、というのが確認できます」

---

### STEP 5: 銀行格付シミュレーション（5分）

**画面：** 経営シミュレーター内の「銀行格付シミュレーション」セクション

#### 説明ポイント

> 「銀行が融資の審査で見ている指標を、129点満点でスコアリングしています。
> これは実際の銀行の内部格付と同じ考え方で作っています」

#### 4つのカテゴリ

| カテゴリ | 見ているもの |
|---------|-----------|
| 安全性 | 自己資本比率・ギアリング比率 → 「倒産しにくいか」 |
| 収益性 | 利益率・ROA → 「ちゃんと稼いでいるか」 |
| 規模・成長性 | 売上高・自己資本額 → 「会社の規模と成長力」 |
| 返済能力 | 債務償還年数・CF額 → 「借りたお金を返せるか」 |

#### ランクの説明

| ランク | 点数目安 | 銀行の見方 |
|-------|---------|----------|
| A（100点以上） | 優良 | 正常先。金利優遇の交渉も可能 |
| B（80〜99点） | 良好 | 正常先。標準的な融資条件 |
| C（60〜79点） | 普通 | 要注意先予備。条件付きで融資可能 |
| D（40〜59点） | 要注意 | 要管理先。追加融資は困難 |
| E（40点未満） | 危険 | 破綻懸念先。融資回収の対象 |

> 「今のシミュレーション結果だと○ランクです。
> ここから売上を10%伸ばして、借入返済を進めると…（スライダー操作）…○ランクに上がります。
> こうやって、銀行にどう見られるかを事前に確認しながら経営計画を立てられます」

## 3. クロージングトーク（3分）

### 導入メリットのまとめ

> 「まとめると、AI CFOでできることは3つです。
>
> 1つ目、**経理の自動化**。レシートを撮影するだけでfreeeに自動登録。経理の時間を削減。
>
> 2つ目、**経営の見える化**。今の数字をベースに未来をシミュレーション。根拠のある計画が作れる。
>
> 3つ目、**銀行評価の事前確認**。融資を受ける前に、銀行からどう評価されるかがわかる。
> 格付を上げるために何をすべきかも明確になる。
>
> 経営者の時間を経理作業から解放し、経営判断に集中できる環境を作ります。」

### 料金の話の前に

> 「まずは御社のfreeeのデータを取り込んで、実際の数字でシミュレーションを回してみませんか？
> 初回のデータ接続とシミュレーション体験は無料で行います」

## 4. デモ前の準備チェックリスト

- [ ] サーバーが起動していること（`http://localhost:3000` にアクセスできる）
- [ ] freeeに接続されていること（事業所が選択済み）
- [ ] デモ用のレシート画像を2〜3枚用意
- [ ] KPI目標にデモ用の数値が入っている
- [ ] ブラウザのウィンドウを大きめに開いておく（グラフが見やすいように）
- [ ] デモモードの場合：「設定 → デモモード → プロファイル選択」で業種を合わせる

## 5. トラブル対応

| 症状 | 対応 |
|------|------|
| ページが表示されない | `localhost:3000` でサーバーが起動しているか確認 |
| freee実績が表示されない | 「設定 → 事業所の選択」でfreee連携を確認 |
| レシート読み取りが失敗 | 画像がぼやけていないか確認。明るい場所で再撮影 |
| グラフが表示されない | ブラウザをリロード（Ctrl+Shift+R） |
| 銀行格付が表示されない | 経営シミュレータータブを再クリック |
